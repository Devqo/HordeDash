# Contributing to HordeDash

Thank you for your interest in contributing to HordeDash! We welcome contributions of all kinds, including bug reports, feature requests, documentation improvements, and code contributions.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally: `git clone https://github.com/YOUR_USERNAME/HordeDash.git`
3.  **Create a new branch** for your feature or bug fix: `git checkout -b feature/your-feature-name` or `bugfix/issue-description`
4.  **Install dependencies** using the provided scripts (`setup.cmd` or `setup.sh`).

## Submitting Pull Requests

*   Keep your Pull Requests focused and small.
*   Write clear, concise commit messages.
*   If your PR fixes an open issue, mention it in the description (e.g., `Fixes #123`).
*   Ensure that your code follows the existing style and conventions.
*   Test your changes locally before submitting.

## Reporting Issues

*   Use the provided issue templates when creating a new issue.
*   Provide as much detail as possible, including OS, Python version, and error logs if applicable.

We look forward to your contributions!
